## 🗺️ Як працює (потік даних)
```
👤 Глядач донатить на Ko-fi
   │  Ko-fi шле webhook (HTTP-POST, JSON: сума, ім'я, verification_token)
   ▼
☁️ Cloudflare Worker — POST /webhook
   1) перевіряє verification_token (це справді наш Ko-fi?)
   2) читає суму з KV  →  3) додає донат  →  4) пише назад у KV
   ▼
🗄️ Cloudflare KV (ключ = id стрімера → {current, goal})
   ▼
📊 Goal Bar в OBS — goal.html
   • кожні ~7 сек: GET /total?id=…  →  Worker віддає {current, goal}
   • смужка плавно оновлюється САМА
```

## 🧩 Компоненти
| Компонент | Роль |
|---|---|
| **Cloudflare Worker** | `POST /webhook` (приймає донат, захищено токеном) + `GET /total` (віддає суму, публічне) |
| **Workers KV** | сховище: ключ = id стрімера, значення `{current, goal}` |
| **Ko-fi** | донат-платформа; webhook + Verification Token |

## 🔧 Зміни (зроблено)
- **Віджет `goal.html`:** є `?api=<worker>/total` → опитує що ~7 с і оновлює смужку. Без `api` → ручний режим (v1), не ламається.
- **Конструктор:** перемикач **«живий режим»** → сам додає `api=` у лінк.

## ✅ Вирішені проблеми (історія)
- **Баг «віджет показує $0» (23.06).** Живий віджет не оновлювався в усіх браузерах. Причина: суворий CSP `default-src 'none'` **тихо блокував** власний `fetch` (помилка в Console: «Refused to connect… violates CSP»). **Фікс:** додано `connect-src https:` у CSP `goal.html`. Урок записаний у `БЕЗПЕКА_чеклист-продуктів.md`.
- **Тест-донат не додавався.** Токен не збігався, поки не зробили **Rotate** у Ko-fi dashboard → після цього `token_match: true`, сума додалась. Діагностували через `wrangler tail`.
